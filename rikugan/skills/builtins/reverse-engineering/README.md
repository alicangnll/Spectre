# Reverse Engineering Mastery

Comprehensive reverse engineering analysis covering binary analysis, decompilation, assembly understanding, control flow analysis, data flow analysis, and reconstruction techniques.

## When to Use

Use this skill when:
- Analyzing binary executables and understanding program structure
- Reconstructing high-level logic from assembly code
- Performing comprehensive control flow and data flow analysis
- Understanding compiler optimizations and code patterns
- Reversing proprietary algorithms and data structures
- Analyzing obfuscated code and anti-debugging techniques
- Documenting reverse engineering findings
- Creating detailed technical analysis reports

## Capabilities

### Binary Analysis
- **Executable Structure**: Analyze PE, ELF, Mach-O file formats and sections
- **Entry Points**: Identify main entry points, initialization routines, startup code
- **Section Analysis**: Understand code, data, import/export sections and their purposes
- **Binary Headers**: Parse and interpret header information and compilation artifacts
- **Architecture Detection**: Determine target architecture, calling conventions, ABI

### Decompilation & Analysis
- **Assembly Understanding**: Explain assembly instructions, patterns, and idioms
- **Control Flow Reconstruction**: Build CFGs, identify basic blocks, execution paths
- **Data Flow Analysis**: Track variable usage, data transformations, dependencies
- **Pattern Recognition**: Identify common compiler patterns, library functions, algorithms
- **Type Recovery**: Reconstruct data types, structures, function signatures
- **Pseudo-code Generation**: Convert assembly to readable pseudo-code

### Advanced Techniques
- **Obfuscation Analysis**: Handle packed/encrypted binaries, code obfuscation
- **Anti-Debugging**: Detect and analyze anti-debugging and anti-analysis techniques
- **Virtualization**: Analyze virtualized code, custom VM implementations
- **Code Optimization**: Understand compiler optimizations and de-optimization
- **Binary Diffing**: Compare binary versions, identify changes and patches
- **Reconstruction**: Reconstruct high-level algorithms and data structures

### Documentation
- **Technical Reports**: Create detailed analysis documentation
- **Flow Diagrams**: Generate control flow and data flow diagrams
- **Annotation**: Document findings, hypotheses, and conclusions
- **Evidence**: Provide concrete examples and proof for claims

## Workflow

1. **Binary Overview**
   - Identify file type, architecture, compiler
   - Map out section structure and layout
   - Find entry points and main execution paths

2. **Function Analysis**
   - Build control flow graphs
   - Analyze function boundaries and calling patterns
   - Identify key algorithms and data structures

3. **Data Flow**
   - Track variable lifetimes and usage
   - Map data transformations
   - Identify global data access patterns

4. **Pattern Recognition**
   - Match known code patterns and idioms
   - Identify library functions and standard algorithms
   - Detect custom implementations and optimizations

5. **Documentation**
   - Document findings with concrete examples
   - Create technical analysis reports
   - Provide evidence-backed conclusions

## Output

Detailed reverse engineering analysis including:
- Binary structure and layout analysis
- Function-level control flow and data flow
- Reconstructed algorithms and data structures
- Pattern recognition and compiler analysis
- Technical documentation with evidence
- Actionable findings and recommendations

Focus on providing concrete, evidence-based analysis with assembly examples and reconstructed logic.
